# M4.1.1 Run M3.5 Hotfix

**Date**: 2026-06-24
**Scope**: Behavior-preserving hotfix for `scripts/run_m3_5_postprocessing.py`

## Root Cause

M4.1 extracted `add_value_change_features(df, shifts)` into `src/lead/features.py`
and migrated the main M3 runners, but `scripts/run_m3_5_postprocessing.py`
was left importing the helper through `run_m3_4_ensemble`.

Before extraction, `run_m3_4_ensemble` had a local one-argument wrapper that
closed over the full offline `SHIFTS` set. After extraction, the re-exported
symbol became the lead-owned two-argument helper. The four M3.5 call sites still
passed only the frame, so M3.5 would raise:

```text
TypeError: add_value_change_features() missing 1 required positional argument: 'shifts'
```

This was a verification-net gap: the M4.1 handoff ruff command listed the four
migrated runners but omitted `scripts/run_m3_5_postprocessing.py`.

## Fix

`scripts/run_m3_5_postprocessing.py` now imports lead-owned helpers directly
from `lead`:

+ `BASELINE_FEATURE_COLS`
+ `DOWNSAMPLE_SEEDS`
+ `PROC`
+ `RANDOM_STATE`
+ `SHIFTS`
+ `add_value_change_features`
+ `classification_metrics`
+ `downsample_indices`
+ `load_m3_frame`

Only script-local helpers remain imported from `run_m3_4_ensemble`:

+ `fit_predict_models`
+ `log`

The four one-argument value-change calls were restored to the pre-M4.1 full
offline behavior by passing `list(SHIFTS)`:

+ Current `scripts/run_m3_5_postprocessing.py:191`
+ Current `scripts/run_m3_5_postprocessing.py:192`
+ Current `scripts/run_m3_5_postprocessing.py:392`
+ Current `scripts/run_m3_5_postprocessing.py:393`

## New Guard

Added `tests/test_call_arity.py`.

The guard AST-scans every tracked `scripts/*.py` file and fails if
`add_value_change_features(...)` is called without either a second positional
argument or a `shifts=` keyword. It also includes a tiny synthetic six-row smoke
test proving `add_value_change_features(df, list(SHIFTS))` creates the expected
`lag_value_diff_*` and `lag_value_ratio_*` columns without requiring M3 data.

The guard was confirmed RED against `HEAD`, flagging only:

```text

scripts/run_m3_5_postprocessing.py:190
scripts/run_m3_5_postprocessing.py:390
scripts/run_m3_5_postprocessing.py:391
```

It is GREEN on the fixed working tree.
