# Handoff: M3.3 buds-lab Feature Alignment Complete

**Date**: 2026-06-22
**Status**: Complete - no robust lift over M3.2
**GitHub Issue**: [#15](https://github.com/kuokuant-oss/lead-reproduction/issues/15)
**Canonical line**: 80/20 `building_id % 5 == 4`, offline value-change regime

## Results

M3.3 added the buds-lab feature-alignment set on top of M3.2:

+ Cyclic encodings: `sin/cos(hour, weekday, month)`
+ Weather trailing lags and trailing rolling means for windows `7` and `73`
+ US holiday flag via `holidays`
+ Train-only `gte_site_meter_anomaly`
+ `primary_use + "_" + meter` interaction encoding
+ Site 0 / meter 0 `meter_reading *= 0.2931` correction before value-change

| Run | Features | Val AUC | Precision@0.5 | Recall@0.5 | F1@0.5 | Delta AUC vs M3.2 |
|---|---:|---:|---:|---:|---:|---:|
| M3.2 reference | 137 | 0.9920 | 0.6409 | 0.9665 | 0.7707 | - |
| M3.3 buds-lab alignment | 170 | 0.9913 | 0.6668 | 0.9583 | 0.7864 | -0.0007 |

Interpretation: M3.3 is no-lift/negligible for AUC. It improves threshold-0.5
precision and F1, but not the ranking metric used for the milestone.

Multi-seed real-label AUCs were `0.9913`, `0.9921`, and `0.9916`
(mean `0.9917`, std `0.00034`). One seed clears the M3.2 single-run AUC
`0.9920`, but the mean does not, so this is not a robust lift.

## Sanity Checks

| Check | Result | Interpretation |
|---|---|---|
| Temporal leakage | past-only `0.9906`, future-only `0.9907`, full `0.9913` | Same balanced past/future pattern as M3.2; no new temporal leakage signal. |
| Label-shuffle seeds | `0.5697`, `0.5700`, `0.3905`, `0.4504`, `0.3738` | Not stable at `~0.57`; mean `0.4709`, std `0.0847`. |
| Label-shuffle ablation A | remove `gte_site_meter_anomaly`: `0.5680` | GTE is not the source of seed-42 elevated shuffle AUC. |
| Label-shuffle ablation B | remove GTE + `log_square_feet/year_built/floor_count`: `0.5820` | These building-meta fields are not the single source. |
| Multi-seed real labels | `0.9913/0.9921/0.9916`, mean `0.9917` | M3.3 wraps around M3.2 but does not robustly beat it. |

The seed-42 label-shuffle AUC `0.5697` matches the existing M3.2 pattern
(`0.5669`) and is not introduced by M3.3. The diagnostics point away from
train/validation contamination: building overlap remains `0`, the target encoder
is fit only on train rows and applied to validation, removing GTE does not drop
the shuffle AUC, and shuffle AUC varies widely across shuffle seeds.

## Open Questions

+ Should M3.4 use M3.2 or M3.3 features? Recommendation: use M3.2 as the AUC
  baseline, but keep M3.3 features available if precision/F1 at threshold 0.5
  becomes more important than AUC.
+ Is a deeper label-shuffle source investigation needed? Not before M3.4. The
  M3.3 diagnostics show no new leakage versus M3.2, and the elevated seed-42
  shuffle result is not stable across shuffle seeds.
+ Should weather interpolation/GMT offset be added? Leave out unless M3.4 or
  later work needs a separate weather-alignment ablation; M3.3 already showed no
  robust AUC lift from the priority buds-lab feature group.
