# Handoff: M3.4 4-model Ensemble Complete

**Date**: 2026-06-22
**Status**: Complete - modest positive lift over M3.2
**GitHub Issue**: [#16](https://github.com/kuokuant-oss/lead-reproduction/issues/16)
**Canonical line**: 80/20 `building_id % 5 == 4`, offline value-change regime
**Feature set**: M3.2 137 features

## Summary

M3.4 trained the paper-style 4-model equal-weight ensemble on the canonical M3
line: LightGBM, XGBoost, CatBoost, and HistGradientBoosting over the M3.2
137-feature set. M3.3 was no-lift for AUC, so it was intentionally not used for
the headline ensemble run.

The seed-42 ensemble reaches validation AUC `0.9928`, a `+0.00079` lift over
the M3.2 LightGBM baseline `0.9920` and a `+0.00078` lift over the best seed-42
single model. This clears the prior `0.0005` noise-floor convention, but the
effect remains modest. The main M3 story is still the M3.2 value-change feature
jump; M3.4 adds a small, stable ensemble gain.

## Open Questions

+ Should M3.5 use the M3.4 ensemble as the post-processing base? Recommendation:
  yes. The ensemble is the current best canonical M3 score.
+ Should M3.3 170-feature ensemble be run? Not necessary before M3.5. M3.3 was
  no-lift for LightGBM AUC, and M3.4 already gives a small positive ensemble
  lift on the cleaner M3.2 feature baseline.
+ Why is CatBoost below XGBoost for seeds 42 and 999, but above XGBoost for seed
  123? Treat as model-seed noise unless later work needs per-model diagnostics.
  The ensemble is stable despite this swap.
+ Should CatBoost runtime expectations be revised? In this local run CatBoost
  took ~3.4-3.6 minutes per seed rather than 30-60 minutes, likely due to the
  current hardware and installed binary performance.
