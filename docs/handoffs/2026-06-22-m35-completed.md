# Handoff: M3.5 Post-processing Complete

**Date**: 2026-06-22

**GitHub Issue**: [#17](https://github.com/kuokuant-oss/lead-reproduction/issues/17)
**Canonical line**: 80/20 `building_id % 5 == 4`, offline M3.2 features, M3.4 seed-42 ensemble

## Summary

M3.5 tested whether the LEAD/M2 hard post-processing rules transfer to full
GEPIII. They do not: Rule 1 and Rule 2b together reduce validation AUC by
`-0.000054`, so M3 closes this branch as a null result rather than forcing a
positive rule. The runner now supports `--allow-null` to persist this confirmed
negative outcome.

The final review also documented the main limitations: cross-site generalization
is materially lower (`0.9774` site-held-out ensemble AUC) and steam is the
weakest meter type (`0.9553` AUC). These are limitations to carry into downstream
FDD work, not cleanup items to hide.

The remaining PI-spec gap was filled before closing M3. The 4-model
LGB/XGB/CatBoost/HistGBT equal-weight ensemble was rerun on the requested 50/50
building split (`building_id % 2`, 725/724 buildings, overlap 0) in both offline
and causal regimes.
