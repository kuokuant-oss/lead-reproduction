### M3 Stage 1: Baseline pipeline (17 features)

**Feature set (17 features from raw CSVs)**:

| Group | Features |
|---|---|
| Meter | meter, meter_reading |
| Temporal | hour, weekday, month, dayofyear |
| Building | primary_use_enc, log_square_feet, year_built, floor_count |
| Weather | air_temperature, cloud_coverage, dew_temperature, precip_depth_1_hr, sea_level_pressure, wind_direction, wind_speed |

**Pipeline** (identical to M2 pattern):
- Train/val split: `building_id % 5 == 4` → val (ADR 0001)
- Downsampling: 50:50 negs1+pos+negs2+pos, random_state=10/20
- StandardScaler → LightGBM(n_estimators=100, random_state=42)

   - Option: vectorized shift on sorted DataFrame (faster than per-building loop)

2. **ClusterNo**: 1449 buildings × full year timestamps → larger pivot; feasible?
   - M2 pivot was 8784 × 406; M3 would be ~8784 × 1449 (manageable)

3. **SavGol**: per-building loop on 1449 buildings — runtime estimate ~7× M2

4. **Rule 2a re-design**: M2 Rule 2a used building_id > 145 OR < 105 (LEAD-specific);
   M3 building_ids run 0–1448, need different filter or skip Rule 2a

   Training time for CatBoost 1000 iters on 4.3M rows: estimate 30–60 min.
