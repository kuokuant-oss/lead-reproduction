## M5 Entry Criteria

+ Data interface: reuse `load_m3_frame`-style loaders for tabular frames. BDG2
  ingestion belongs to M5, not M4.
+ Label interface: BDG2 may have no per-row anomaly labels, so M5 must allow
  unlabeled or transfer evaluation instead of assuming a 1:1 anomaly label
  column.
+ Split interface: use building-level and site-held-out splits through
  `split_mask` and `assert_no_building_overlap`. Site-held-out transfer is
  motivated by M3 site-held-out ensemble AUC `0.9774`.
+ Evaluation interface: use `classification_metrics` for AUC, precision,
  recall, and F1. Any real-time FDD claim must use `PAST_SHIFTS`-only causal
  features per ADR 0007 and ADR 0011.
